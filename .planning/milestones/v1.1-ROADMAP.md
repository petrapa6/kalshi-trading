# Milestone v1.1: Local-JSON Backtest

**Status:** ✅ SHIPPED 2026-04-29
**Quick Tasks:** 4
**Total Plans:** 4

## Overview

Replaced the dashboard backtest page's dependency on `/api/backtest/soccer` and Kalshi market prices with a self-contained, deterministic experience driven by pre-fetched season JSONs in `resources/`. The page is now a pure client-side computation with no network calls after page load.

This milestone was executed as a series of quick tasks (rather than formal GSD phases) because the scope was contained to `dashboard/app/backtest/` and moved fast.

## Quick Tasks

### Quick Task 260429-h5z: Local-JSON Backtest Page (BT-03..05)

**Goal**: Replace API+price wiring with a local JSON season loader
**Requirements**: BT-03, BT-04, BT-05
**Status**: ✅ complete (commits 9f7dd67, 01381c6)

Plans:
- [x] 260429-h5z-01: Refactor backtest page — seasons catalog + pure strategy engine

**Details:**

Created `dashboard/app/backtest/seasons.ts` (build-time season catalog, `SeasonFile` / `Match` / `Goal` types, `parseSeasonFilename`, `SEASONS` array with static imports) and `dashboard/app/backtest/backtest.ts` (pure strategy engine: `parseGoalTime`, `parseScore`, `detectFire`, `runBacktest` — no prices). Rewrote `page.tsx` to use a `SEASONS` dropdown, reactive sliders, and removed the API fetch, recharts `BankrollChart`, and all Kalshi-price controls.

Key decision: used static imports over `import.meta.glob` — Next.js 16 Webpack does not expose `import.meta.glob` without a custom loader; static imports bundle JSON at build time and work under `output: standalone`.

## Milestone Summary

**Key Decisions:**

- Static-import catalog over `import.meta.glob` — Next.js 16 Webpack doesn't expose glob without a plugin; static imports satisfy `output: standalone` and bundle JSON at build time.
- Keep `/api/backtest/soccer` endpoint intact — backend may still be invoked from CLI/scripts; only dashboard wiring changes.
- Quick tasks over formal GSD phases — scope was contained and moved fast; no multi-step planning required.

**Issues Resolved:**

- `FILENAME_RE` extended for calendar-year league filenames (MLS `_2025_` pattern).
- `LEAGUE_NAMES` keys corrected (`laliga` not `pd`, `bundesliga` not `bl1`).
- All 6 season JSONs wired into static import catalog.

**Issues Deferred:**

- Hand-maintained `IMPORTS` array in `seasons.ts` — requires editing for each new season (documented TODO).
- `simulateMatch` shim retained as back-compat wrapper around `detectFire`.
- No fee or partial-fill modeling in backtest engine.
- Pre-existing `pnpm fmt:check` failures in `app/page.tsx`, `app/actions.ts`, `app/api/[...path]/route.ts`, `sst-env.d.ts` — predates this milestone.

**Technical Debt Incurred:**

- `avgWinYield` input range (0.001..1) is very wide; no validation rules tighten it.
- Catalog expansion in `260429-k6u` committed before the 3 other JSONs from chore `4608cf3` were added — resolved in follow-on.
