## Directory Purposes

**`.planning/`:**
- Purpose: Planning documents, phase states, UAT checklists
- Contains: PROJECT.md, STATE.md, phases/* (phase plans, summaries, reviews)
- Generated by: `/gsd-plan-phase`, `/gsd-map-codebase`
- Committed: Yes

**`src/predictions/`:**
- Purpose: Python backend package (installed editable via hatchling)
- Contains: FastAPI app, scanner loops, DB models, Kalshi/ESPN clients
- Key files: `api.py` (endpoints), `scanner.py` (core logic), `db.py` (schema)
- Committed: Yes

**`tests/`:**
- Purpose: Python test suite (async via pytest-asyncio)
- Contains: Standalone scripts exercising scanner loops, WS connection, sport stats
- Not integrated with CI; run manually: `uv run pytest tests/`
- Committed: Yes

**`cli/`:**
- Purpose: React-ink TUI for local config/stats access
- Contains: Meow arg parser, API client, view components
- Built: `pnpm build:cli` (if target added; currently invoked via `pnpm cli`)
- Committed: Yes; includes src/

**`dashboard/`:**
- Purpose: Next.js 16 read-only SPA deployed on production
- Contains: Single monolithic `page.tsx` component (known maintainability issue)
- Built: `pnpm build` (generates `.next/` folder)
- Committed: Yes; includes app/, but not `.next/` or `node_modules/`

**`scripts/`:**
- Purpose: Pre-commit hook and dev utilities
- Key: `pre-commit-check.sh` runs ruff + ty; installed as git hook by `./install.sh`
- Committed: Yes

**Root config files:**
- `pyproject.toml` — Python package metadata, ruff/ty/pytest config, hatchling build backend
- `package.json` — pnpm root workspace scripts (dev, sst:deploy, lint, pre-commit-check)
- `sst.config.ts` — SST infra: ECS Fargate, S3, Cloudflare DNS, secrets
- `.env.example` — Canonical env schema (copy to `.env` locally)

## Key File Locations

**Entry Points:**
- API: `src/predictions/api.py:app` (FastAPI instance; spawns scanner on startup)
- Scanner: `src/predictions/scanner.py:run_scanner()` (async main loop; spawned by API lifespan)
- CLI: `cli/src/index.tsx` (meow arg parser + App router)
- Dashboard: `dashboard/app/page.tsx` (Next.js page component; 102 KB monolith)

**Configuration:**
- Runtime config: SQLite `config` table (K-V store); read via `get_config_int()` in `db.py`
- Env config: `.env` (copy of `.env.example`); never committed
- Build config: `pyproject.toml` (Python), `next.config.ts` (Next.js), `sst.config.ts` (infra)
- Lint/format: `.prettierrc` missing (rely on oxfmt.json in dashboard), ruff config in pyproject.toml

**Core Logic:**
- Trade placement: `src/predictions/scanner.py:place_bet()` (lines 127–199)
- Market matching: `src/predictions/scanner.py:scan_kalshi_with_espn()` (lines 300–576)
- Settlement: `src/predictions/scanner.py:on_lifecycle()` handler (lines 825–877) and `check_settlements()` (lines 202–238)
- ESPN polling: `src/predictions/scanner.py:espn_loop()` (lines 882–907)
- Kalshi discovery: `src/predictions/scanner.py:kalshi_scan_loop()` (lines 909–1000)
- What-if evaluation: `src/predictions/scanner.py:_evaluate_what_if_strategies()` (lines 579–704)

**Testing:**
- Scanner tests: `tests/test_sport_stats.py`, `tests/test_ws.py` (standalone scripts, not integrated)
- API endpoints: No unit tests; integration testing via `/gsd-qa` during phases

**Utilities:**
- Kalshi client: `src/predictions/kalshi_client.py` — RSA signing, rate limiting, price extraction
- ESPN client: `src/predictions/espn.py` — Game state, team matching, final-minutes detection
- DB helpers: `src/predictions/db.py:get_config_int()`, `get_session()`, `init_db()`
- Backtest engine: `src/predictions/backtest.py` — What-if scenario evaluation

## Naming Conventions

**Python Files:**
- Module-level: `snake_case.py` (e.g., `kalshi_client.py`, `espn.py`)
- Classes: `PascalCase` (e.g., `Trade`, `GameState`, `KalshiClient`)
- Functions: `snake_case` (e.g., `extract_cents()`, `get_scoreboard()`)
- Constants: `UPPER_SNAKE_CASE` (e.g., `MIN_VOLUME`, `WHAT_IF_STRATEGIES`)

**TypeScript/JavaScript Files:**
- Components: `PascalCase.tsx` (e.g., `ConfigView.tsx`, `StatsView.tsx`)
- Utilities: `camelCase.ts` (e.g., `api.ts`)
- Files: `kebab-case` for Next.js special files (e.g., `[...path].ts`)

**Database Tables:**
- Plural: `trades`, `opportunities`, `stretch_opportunities`, `scans`, `balance_snapshots`, `config`
- Columns: `snake_case` (e.g., `yes_price`, `espn_clock_seconds`, `pnl_cents`)

**API Endpoints:**
- Pattern: `/api/<resource>` (e.g., `/api/stats`, `/api/trades`, `/api/config`)
- Query params: `camelCase` (e.g., `?limit=50&offset=0`)

**Environment Variables:**
- Pattern: `UPPER_SNAKE_CASE` (e.g., `API_TOKEN`, `KALSHI_API_KEY`, `DATABASE_URL`)
- Sensitive: Never committed; stored in `.env` (local) or SST secrets (production)

## Where to Add New Code

**New Backend Endpoint:**
- File: `src/predictions/api.py`
- Pattern: Define Pydantic response model, then `@app.get/post()` handler with `dependencies=[Depends(_check_token)]` for mutations
- Example: Adding `/api/stretch-stats` for what-if summary

**New Scanner Loop or Refactoring Existing Logic:**
- File: `src/predictions/scanner.py::run_scanner()`
- Pattern: Define async function (e.g., `async def new_loop()`), spawn via `asyncio.gather()` at line 1019
- Example: Adding a new polling interval (currently ESPN 10s, Kalshi 5s, WS listen, backup 30m)

**New Dashboard Feature or Page:**
- File: Break out from `dashboard/app/page.tsx` into sub-components in `dashboard/app/components/` (not yet created)
- Pattern: React functional component, use `useEffect` + fetch for data; leverage existing `/api/[...path]` proxy for Bearer auth
- Example: Adding a config editor panel alongside stats

**New CLI Command:**
- File: `cli/src/index.tsx` (meow routing) + new component in `cli/src/components/`
- Pattern: Add case in `App` switch statement; create `<YourCommand>` ink component
- Example: Adding `getrich backtest --sport soccer` for local backtesting

**New Database Model or Schema Migration:**
- File: `src/predictions/db.py`
- Pattern: Define SQLAlchemy `Base` subclass with columns; add inline migration in `_migrate_add_columns()` using `ALTER TABLE`
- Example: Adding `match_id` column to `Trade` for external cross-referencing

**Shared Utilities:**
- File: Existing modules (`kalshi_client.py`, `espn.py`, `db.py`) or new `utils.py` if truly reusable
- Pattern: Pure functions with clear inputs/outputs; no side effects if possible
- Example: New price formatting function alongside `extract_cents()`

**Test Addition:**
- File: `tests/test_<feature>.py`
- Pattern: Use `pytest` + `pytest-asyncio` for async tests; mark with `@pytest.mark.asyncio`
- Run: `uv run pytest tests/`

## Special Directories

**`.sst/`:**
- Purpose: SST build artifacts (CloudFormation, deployment metadata)
- Generated: By `sst dev` and `sst deploy`
- Committed: No (in .gitignore)

**`.next/`:**
- Purpose: Next.js build output
- Generated: By `pnpm build` in dashboard/
- Committed: No (in .gitignore)

**`__pycache__/` and `.pytest_cache/`:**
- Purpose: Python compilation cache and pytest metadata
- Generated: By Python interpreter and pytest runner
- Committed: No (in .gitignore)

**`node_modules/`:**
- Purpose: npm/pnpm installed packages
- Generated: By `pnpm install`
- Committed: No (in .gitignore); use lockfiles (pnpm-lock.yaml, uv.lock)

**`predictions.db`:**
- Purpose: Local SQLite database (development only)
- Generated: By `src/predictions/db.py:init_db()` on first run
- Committed: No (in .gitignore); production uses `/tmp/predictions.db` in ECS + S3 backup

**`scanner.log`:**
- Purpose: Scanner loop output
- Generated: By Python `logging.FileHandler` in `scanner.py`
- Committed: No (in .gitignore); rotated via container restart in ECS
