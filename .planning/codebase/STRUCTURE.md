## Naming conventions

- Python modules: snake_case (`kalshi_client.py`, `soccer_cache.py`).
- Python classes: PascalCase (`KalshiClient`, `BacktestRequest`).
- Python functions: snake_case; private prefix `_` for module-private (`_check_token`, `_download_db`, `_evaluate_what_if_strategies`).
- TypeScript components: PascalCase (`StatCard`, `LoginForm`, `LiveGamesPanel`).
- DB tables: snake_case plural where applicable (`trades`, `opportunities`, `stretch_opportunities`, `balance_snapshots`, `config`, `scans`, `soccer_matches`, `soccer_goals`).
- Config keys: dotted/colon-namespaced strings (`min_yes_price`, `lead:basketball/nba`, `final_seconds:soccer/eng.1`).
- Kalshi series tickers: uppercase prefix `KX…` (`KXNBAGAME`, `KXMLBGAME`).

## Where to put what

- New endpoint → `src/predictions/api.py`, with a Pydantic response model in the same file.
- New scanner behavior → extend functions in `src/predictions/scanner.py`. New runtime tunables → add to `_CONFIG_DEFAULTS` in `src/predictions/db.py`, then read with `get_config_int`.
- New external integration → its own module under `src/predictions/`, mirror the `kalshi_client.py` pattern (single drift point).
- New DB column → add to the model in `src/predictions/db.py`, then add an `ALTER TABLE` block in `_migrate_add_columns()` for production deployments where the table already exists.
- New dashboard view → currently must land inside the monolith `dashboard/app/page.tsx` or as a sibling route under `dashboard/app/<name>/page.tsx` (see `backtest/page.tsx` for the pattern).
- New CLI subcommand → register the route in `cli/src/app.tsx` and add a component under `cli/src/components/`.
- New test → `tests/test_<thing>.py`. Conftest gives you `isolated_db` and `isolated_soccer_db` automatically.
