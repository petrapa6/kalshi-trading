## Filename-parser Regex

Regex shipped: `^([a-z0-9]+)_(\d{4})_(\d{2})_season\.json$`

- Captures: league (lowercase alphanumeric), 4-digit start year, 2-digit end year.
- End year resolved to 4-digit via `Math.floor(startYear / 100) * 100 + endYY`.
- Non-matching filename: returned as `null` from `parseSeasonFilename`, filtered out via
  `flatMap` in the SEASONS builder, and logged with `console.warn` in non-production
  environments. The catalog proceeds with remaining valid entries — no crash.

## import.meta.glob vs Static-Import Fallback

**Static-import fallback used.**

`import.meta.glob` is a Vite/webpack-5 plugin feature. Next.js 16 ships Turbopack for dev
and Webpack for production builds, but neither exposes `import.meta.glob` without a custom
loader plugin. Attempting to use it would compile (TypeScript accepts it as `any`) but
produce an empty object at runtime, resulting in `SEASONS.length === 0` even when files
exist.

The plan's fallback path was taken: a hand-maintained `IMPORTS` array holds
`{ filename, data }` pairs with `import epl_2024_25 from "../../../resources/epl_2024_25_season.json"`.
Webpack bundles the JSON with the client component at build time. This also satisfies
`output: "standalone"` — the standalone trace follows the static import and includes the
JSON in the bundle.

A TODO comment in `seasons.ts` documents that adding a season requires editing the IMPORTS
array.
